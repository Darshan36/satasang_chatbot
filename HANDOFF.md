# Akshar Sarjan Prasang Explorer - Handoff Document

## Project Overview
A web explorer for spiritual stories (Prasangs) extracted from Telegram chat exports.
Search works **semantically**: every story is embedded with Gemini, and a typed query is
matched by *meaning* (not keyword buckets), so results are ranked by genuine relevance.
Users can also **browse by topic** using an AI-derived taxonomy of topics.

## System Architecture
- **Backend:** Flask (Python), raw `requests` (no SDKs).
- **Frontend:** Vanilla JS / CSS (responsive, blue `#1a237e` / gold `#ffd700`).
- **Embeddings (search):** Gemini `gemini-embedding-001` (768-dim, L2-normalized) for
  stories, topics, and the live query.
- **Categorization (tags/chips):** Groq `llama-3.3-70b-versatile` (JSON mode) — Groq's
  free tier has a high daily request budget, ideal for tagging all 453 stories.
- **Taxonomy:** Gemini `gemini-2.5-flash-lite` — one call to derive the topic list.
- **Search math:** cosine similarity (numpy) over the in-memory embedding matrix.

> Why two providers: Groq has no embeddings endpoint (LLM/chat only), and Gemini's free
> *generation* quota is tiny. So generation-heavy tagging runs on Groq; embeddings run on
> Gemini (a one-time, resumable index build + 1 small call per live search).

## How search works
- **Typed query →** embedded at runtime (`RETRIEVAL_QUERY`) → cosine vs all story vectors →
  top-K ranked stories, each shown as its own card with a "% match" badge.
- **Topic chip →** in-memory filter on the story's `topics`, ordered by closeness to that topic.
  No API call, instant.

### Query expansion (cross-script matching)
Stories are in Gujarati script but users type romanized terms (e.g. "kartaharta"), which
embeddings match poorly — and an LLM can mis-transliterate niche terms (kartaharta → karyakarta).
So `app.py`:
1. checks `glossary.json` — if the query contains a known satsang term, it appends a **curated,
   authoritative expansion** (Gujarati script + meaning) before embedding;
2. otherwise embeds the raw query, and only if the best match is weak (< `SIM_THRESHOLD`) does it
   ask Groq to enrich the query as a rescue (keeps clear queries crisp).

**To make a new term searchable, add it to `glossary.json`** (`aliases` = trigger phrases,
`expansion` = Gujarati + English meaning). This is the main knob for fixing "X doesn't find the
right stories".

## Key Files
### App
- `app.py` — Flask server. `/ask` handles both `{query}` (semantic) and `{tag}` (browse) modes.
- `templates/index.html` — UI: search box, topic chips, ranked per-story cards.
- `glossary.json` — curated romanized→Gujarati expansions for key satsang terms. **Editable.**
- `config.py` — loads `GEMINI_API_KEY` from `.env`; model names + embedding dim.
- `gemini_api.py` — `generate_json()` and `batch_embed()` REST helpers (retry honors 429 `retryDelay`).
- `clean_text.py` — strips Date/Day/Place/Time/Topic/Prasad metadata; `content_hash()`.

### Data (generated by the pipeline, all index-aligned)
- `granular_stories.json` — raw source: 504 split stories `{session_topic, content}`.
- `stories_base.json` — canonical cleaned + de-duplicated stories (defines row order).
- `topics.json` — the AI-derived topic taxonomy `{name, description, synonyms}`. **Editable.**
- `recategorized_stories.json` — **the DB the app serves**: `{session_topic, content, topics:[...]}`.
- `embeddings.npy` — `(N, 768)` float32 normalized story vectors (aligned to stories_base/recategorized).
- `topic_embeddings.npy` — `(T, 768)` normalized topic vectors.
- `embeddings_meta.json` — model/dim/counts + per-row content hashes (mismatch detection + reuse).

## Data Pipeline (recreate / update)
Run in order from the project directory:
1. **`python build_taxonomy.py`** → `topics.json` (2-pass: extract themes → consolidate ~25-40 topics).
   *Review/edit `topics.json` before continuing.*
2. **`python build_embeddings.py`** → `stories_base.json`, `embeddings.npy`, `topic_embeddings.npy`,
   `embeddings_meta.json`. Re-running only embeds new/changed stories (hash cache).
3. **`python recategorize_groq.py`** → `recategorized_stories.json` (Groq tags each story
   against the taxonomy; resumes by hash; saves after every batch).
4. **`python app.py`** → serve at `http://localhost:8080`.

> `assign_tags.py` is an alternative tagger (by embedding similarity, no Groq) kept as a
> fallback; the app uses the Groq output by default.
> Set both `GEMINI_API_KEY` and `GROQ_API_KEY` in `.env`.

## API key & security

- Keep the UI clean; metadata (Date/Time/Location) stays hidden — now actually enforced by
  `clean_text.clean_content()` (the source still contained those headers).
- Users prefer specific stories over full session summaries (one card = one story, ranked).
- Maintain Primary Blue (`#1a237e`) and Accent Gold (`#ffd700`).

## Tuning notes for the next agent
- Search relevance: `TOP_K`, `SIM_THRESHOLD`, `TAG_BOOST` in `app.py`.
- Tagging coverage: `TAG_THRESHOLD`, `KEEP_FLOOR`, `MAX_TAGS` in `assign_tags.py` (`--stats` first).
- To add/rename topics, edit `topics.json`, then rerun steps 2-3 (only topic embeddings + tags change).
